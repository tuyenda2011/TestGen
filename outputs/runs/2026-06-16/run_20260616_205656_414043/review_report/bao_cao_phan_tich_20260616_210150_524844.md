## Rủi ro maintainability
- **Phụ thuộc vào môi trường cục bộ**: URL được hard‑code trong biến `base_url` (`http://localhost:8000`). Nếu môi trường thay đổi (cổng, host), cần chỉnh sửa collection hoặc biến môi trường.  
- **Không có dữ liệu mock**: Test thực hiện gọi thật tới server đang chạy. Nếu server không sẵn sàng hoặc dữ liệu thay đổi, test sẽ thất bại (flaky).  
- **Thiếu cấu hình timeout / retry**: Newman mặc định có timeout ngắn; nếu API chậm, test có thể báo lỗi không phải do logic.
