## Lỗi nghiêm trọng
- **TC-001:** Assertion kiểm tra `body.total` so sánh với giá trị **100.5**, trong khi yêu cầu (outputs) chỉ định `totalBeforeCredit = 110.5` và không yêu cầu trường `total`.  
  - **Bằng chứng:**  
    ```json
    // Requirement output TC-001
    "diagnostics": {
      "discount": 12,
      "shipping": 22.5,
      "totalBeforeCredit": 110.5
    }
    ```  
    ```javascript
    // Script trong collection
    pm.test("response has expected total", function () {
      const body = pm.response.json();
      pm.expect(body.total).to.eql(100.5);
    });
    ```
  - **Ảnh hưởng:** Kiểm thử đang xác nhận một giá trị không tồn tại trong spec, có thể che giấu lỗi tính toán thực tế và làm sai lệch độ tin cậy của bộ kiểm thử.  
  - **Cách sửa:** Xóa assertion `body.total` hoặc thay thế bằng kiểm tra `body.diagnostics.totalBeforeCredit` so sánh với **110.5**.

- **TC-002, TC-004, TC-005:** Yêu cầu yêu cầu trả về **đối tượng lỗi** có hai trường `error` và `message` với nội dung cụ thể, nhưng script chỉ kiểm tra `body.error` có chứa một từ khóa.  
  - **Bằng chứng:**  
    ```javascript
    pm.test("response explains missing subtotal", function () {
      const body = pm.response.json();
      pm.expect(body.error).to.include("subtotal");
    });
    ```
    Yêu cầu:
    ```json
    "error": "validation error",
    "message": "subtotal is required"
    ```
  - **Ảnh hưởng:** Kiểm thử không phát hiện sai lệch nội dung thông báo chi tiết (ví dụ: sai typo, ngôn ngữ, hoặc thiếu trường `message`).  
  - **Cách sửa:** Thêm các assertion:
    ```javascript
    pm.expect(body.error).to.eql("validation error");
    pm.expect(body.message).to.eql("subtotal is required");
    ```
