## Lỗi nghiêm trọng
- **TC-002**: Kiểm thử “POST /posts - missing userId” **không phù hợp với yêu cầu**.  
  - Yêu cầu & Test Plan (TC‑002) yêu cầu trả về lỗi 400/422 khi thiếu `userId`.  
  - Test hiện tại lại kiểm tra **mã trạng thái 201** và khẳng định phản hồi không có `userId`, điều này trái với quy tắc nghiệp vụ và sẽ luôn thất bại (hoặc đưa ra kết quả sai nếu API thực tế trả về 201).  
  - Cần sửa lại để kiểm tra mã lỗi và không kỳ vọng thành công.

## Test còn thiếu
- Không có kịch bản kiểm thử **negative** cho `POST /posts` khi `userId` không phải là số nguyên (ví dụ: `"userId": "abc"`).  
- Không có kiểm thử **boundary** cho giá trị `userId` lớn (ví dụ: `userId: 999999`).  
- Không có kiểm thử **security** xác nhận API không chấp nhận các header không mong muốn hoặc các payload có trường không hợp lệ.  

## Assertion yếu
- Không phát hiện.

## Rủi ro maintainability
- Tất cả các test phụ thuộc vào **API công cộng JSONPlaceholder** trên mạng. Nếu mạng chậm, API tạm ngưng hoặc có thay đổi, các test sẽ **flaky** và gây lỗi không phải do code.  
- Các test “clarification” (TC‑007, TC‑008) không thực sự kiểm tra hành vi chức năng mà chỉ dùng để thu thập thông tin; nếu để trong bộ test tự động sẽ gây nhiễu và không có giá trị kiểm thử.

## Gợi ý sửa ngay
1. **Sửa TC-002**  
   ```json
   // Thay đổi phần kiểm tra:
   "pm.test(\"Status code is 400 or 422\", function () {",
   "    var status = pm.response.code;",
   "    pm.expect([400, 422]).to.include(status);",
   "});"
   // Loại bỏ các kiểm tra về `userId` trong phản hồi.
   ```
2. **Thêm test negative cho `userId` không hợp lệ** (ví dụ `userId: "abc"`), kiểm tra mã lỗi 400/422.  
3. **Thêm test boundary cho `userId` lớn** (ví dụ `userId: 999999`), xác nhận API vẫn trả về 201 và phản hồi chứa đúng `userId`.  
4. **Cân nhắc tách các kịch bản “clarification” ra khỏi bộ test tự động** hoặc chuyển chúng thành tài liệu, để tránh gây nhiễu.  
5. **Giảm phụ thuộc mạng**: có thể dùng mock server (ví dụ: `wiremock`) cho các test quan trọng, hoặc chạy test trong môi trường CI có kết nối ổn định.
